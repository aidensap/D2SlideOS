# 实习答辩思路

## 核心原则

- 讲"我"，不讲"项目细节"
- 听众来自不同业务线，不需要懂技术/业务逻辑
- 每个故事用同一个公式：**痛点 → 我做了什么 → 结果/价值**
- 业务细节一句话带过，结论说清楚

---

## 两条主线

### 线一：Business Consulting 视角
**核心主题**：从零理解企业级 SAP 项目的复杂性

**项目1：MMG 五矿（ECC → S/4 Migration）**
- 痛点（对外说法）：客户系统老旧、有大量自研功能，系统升级的同时要保证业务不中断
- 我的角色：参与项目会议、系统测试
- 要讲的结论：
  > "我第一次理解了企业级项目的复杂性不在于技术本身，而在于如何在系统升级的同时保证业务不中断。顾问的价值不只是写代码，而是帮客户管理风险。"
- 待补充细节：某次会议的核心讨论点 / 测试时发现的问题

**项目2：AIA 友邦保险（费用分摊）**
- 痛点（对外说法）：企业费用分摊逻辑复杂，业务规则多，需要严格的系统验证
- 我的角色：阅读 UR/FS 文档、参与业务会议、做系统测试
- 要讲的结论：
  > "我学会了如何把业务需求翻译成系统逻辑，以及如何通过测试验证系统行为符合业务预期。"
- 待补充细节：某条让你印象深刻的业务规则 / 测试发现的 bug

---

### 线二：AI Innovation 视角
**核心主题**：识别业务痛点 → 独立开发解决方案 → 上线产品

**D2SlideOS**
- 痛点（一句话）：客户觉得 SAC 手动导数据太麻烦，产品价值感知低，导致客户流失
- 我做了什么：独立开发自动化工具，把"手动导数据→分析→做报告"压缩到零人工干预，部署到 SAP BTP 上线
- 结果/价值：
  - 定时自动生成报告发到邮箱，不需要人工操作
  - 接入 SAC OData API，支持 AI 自动分析 + 生成图表 + 生成 PPT
  - 部署在 SAP BTP，任何人可访问，无需本地安装
- 待补充：接入了多少 SAC 模型 / 有没有实际给人演示或使用

**其他 AI 日常工作**
- 帮组里整合客户产品说明 → 知识库
- 日常用 AI 提效（文档整理、信息提炼）
- 待补充：整合了多少份文档 / 帮谁解决了什么具体问题

这句话把两条线串起来，体现**业务洞察 → AI 创新**的因果关系。

## 可能被问到的问题

**业务线相关**
- 你在项目里具体做了什么贡献？
  → 重点讲测试发现的问题，以及从旁听中建立起的业务理解
- Migration 项目最大的难点是什么？
  → 不是技术，是业务连续性——客户不能停业来等系统升级

**AI 线相关**
- 这个工具现在有人在用吗？
  → 待补充实际使用情况
- 为什么选择部署到 BTP 而不是其他平台？
  → SAP 生态内，客户已有 BTP 环境，集成成本最低
- 数据安全怎么保证？
  → SAC OAuth Client Credentials，数据不落地（只在内存处理）
- 如果要商业化，最大的障碍是什么？
  → 用户隔离（目前共用 token）、邮件发送限制、数据库持久化

**成长相关**
- 实习最大的收获是什么？
  → 理解了"顾问"的价值：不只是执行，而是帮客户定义和解决正确的问题
- 遇到最大的挑战是什么？
  → 从学生思维转换到业务思维——学校里解决的是有标准答案的问题，真实项目里需要先把问题定义清楚

## 注意事项

- 不说"我主要旁听"，说"我深度参与了项目会议，理解了..."
- 技术细节（API、CF、SQLite 等）对业务 manager 一句话带过
- 每个故事都要有**具体细节**（哪怕一个），避免全程泛泛而谈
- D2SlideOS Demo 如果现场能打开更好，准备备用截图防止网络问题
